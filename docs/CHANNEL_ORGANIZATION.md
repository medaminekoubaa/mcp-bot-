## 🔒 Channel Restrictions - Technical Details

### How It Works

1. **User types command** in a Discord channel
2. **Bot checks channel name** against allowed commands list
3. **If allowed**: Command executes normally
4. **If not allowed**: User gets helpful error message with correct channel

### Error Message Example

```
❌ Command Not Allowed Here

The command `/dev-update` cannot be used in #general.

📍 Use This Command In:
🔨 Dev Tracking

💬 Commands In This Channel:
`/get-motivation`
```

## 🔧 Managing Channels

### For Admin: How to Pin Channel Info

1. Go to each channel
2. Send the channel info message (bot generates this)
3. Right-click message → Pin message
4. Repeat for all channels

### For Admin: How to Update Commands

If you need to add/remove commands from a channel:

1. Edit `src/services/channelGuard.js`
2. Update the `CHANNEL_POLICIES` object
3. Redeploy bot

Example:
```javascript
'daily-tasks': {
  allowedCommands: [
    'yesterday-summary',
    'today-plan',
    'task-completed',
    // Add new command here
  ],
}
```

## ❓ FAQ

### Q: What if I use the wrong channel?

**A**: You'll get a friendly error message telling you which channel to use.

### Q: Can I move commands to different channels?

**A**: Yes! Edit `channelGuard.js` and redeploy. But it's better to keep the standard structure.

### Q: What about private channels?

**A**: Commands only work in managed channels. Private channels are for off-topic discussion.

### Q: Can I rename the channels?

**A**: The bot looks for channel names, so you can rename them, but keep the logical structure.

### Q: Do all team members need access to all channels?

**A**: Yes, but you can set Discord permissions so certain roles only see certain channels.

### Q: What if we add more commands?

**A**: Assign them to the most appropriate channel or create a new one.
